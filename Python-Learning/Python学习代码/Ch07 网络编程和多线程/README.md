## IP 地址(Internet Protocol Address)

在网络中唯一标识一台计算机或设备的位置。

IP 地址像“家庭地址”，告诉网络“你要把数据包送到哪台机器”。

### IPv4

32 位二进制，通常写作四段十进制：`192.168.1.1`。范围：0.0.0.0 – 255.255.255.255。

分为 **公网 IP** (可在互联网上访问) 和 **私网 IP** (仅在局域网内使用，如 192.168.x.x)。

### IPv6

128 位，解决 IPv4 地址不足问题，写作 8 组 16 进制：`fe80:0000:0001:0000:0440:44ff:1233:5678`。

**分类**：

1、**单播(Unicast)**：一对一通信。

2、**广播(Broadcast)**：一对全体通信。

3、**组播(Multicast)**：一对多通信(特定组)。

## 端口号(Port Number)

标识主机上的应用或服务，相当于房子里的房间号。

同一台电脑上可以有多个程序同时使用网络，通过端口区分服务。

范围：0 – 65535。

**0–1023**：知名端口(HTTP 80，HTTPS 443，SSH 22)。

**1024–49151**：注册端口(数据库、应用程序等)。

**49152–65535**：动态/私有端口(临时通信使用)。

**组合**：IP + 端口 = 套接字地址(Socket Address)，用于唯一标识网络通信端点。

## 协议(Protocol)

通信规则，规定数据如何发送、接收、处理。常见协议主要有 TCP 和 UDP。

### TCP(Transmission Control Protocol)

**特性**：面向连接(先握手再通信)、可靠传输(保证数据不丢失、不重复、有序)、有流量控制和拥塞控制。

**应用场景**：HTTP/HTTPS(网页浏览)、FTP(文件传输)、邮件传输(SMTP/POP3/IMAP)。

TCP 稳妥安全但慢。

#### TCP三次握手

第一次握手：客户端向服务端发送请求，等待服务端确认。

第二次握手：服务端收到请求后知道客户端请求建立连接，回复给客户端以确认连接请求。

第三次握手：客户端收到确认后，再次发送请求确认服务端，服务端收到正确请求后，如果正确则建立成功，完成三次握手，随后客户端与服务端之间开始传输数据。

#### TCP四次挥手

第一次挥手：主机A(可以是客户端，也可以是服务端)完成数据传输后，提出停止TCP连接的请求。

第二次挥手：主机B收到请求后对其做出响应，确认这一方向上的TCP连接将关闭。

第三次挥手：主机B再提出反方向的连接关闭请求。

第四次挥手：主机A确认主机B的请求，双方向的关闭结束。

### UDP(User Datagram Protocol)

**特性**：无连接(发送就发，不保证对方收到)、不可靠(可能丢包、乱序)、开销小速度快。

**应用场景**：DNS 查询、视频直播、在线游戏、广播和组播应用。

UDP快速但不可靠。

## Socket 套接字

Socket 可以理解为程序进行网络通信时使用的“插口”或“接口”。就像电器要通过插头插进插座才能用电一样，程序也需要通过 socket 才能将数据放到网络上传输。

## 进程与线程

进程是操作系统进行资源分配的基本单位，表示一个正在运行的程序实例，拥有独立的地址空间和系统资源。

线程是进程内部的执行单元，是 CPU 调度的基本单位，一个进程至少包含一个线程。

不同进程之间内存相互隔离，而同一进程内的多个线程共享大部分资源(如地址空间和文件描述符)，但各自拥有独立的栈和执行上下文。

进程主要用于运行环境隔离，线程主要用于在同一进程内实现并发执行。

## 多任务

多个任务同时执行能够充分利用CPU资源，大大提高程序执行效率。

多任务指的是在同一时间内执行多个任务(或者给你的感觉是在同时执行多个任务)。

多任务的表现有两种形式：1、并发；2、并行。

并发：在一段时间内，交替执行任务(在很小尺度的时间间隔内，只执行一个任务)。

并行：在一段时间内，真正的同时一起执行任务。
