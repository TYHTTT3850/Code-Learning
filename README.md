# 编程相关概念
## 静态类型与动态类型、强类型与弱类型语言
静态类型：编译时确定变量类型(C、C++、Java、Rust、Go)

动态类型：运行时确定变量类型(Python、JavaScript、Ruby、PHP)

强类型：很少/不允许隐式类型转换(Python、Java、Rust、Go)

弱类型：大量允许隐式类型转换(JavaScript、PHP、Perl)

主流的编程语言分类如下：

![](./0.Figures/分类图.jpg)

python强类型示例：

```python
a = 10
b = "20"
c = a + b       # TypeError，不允许隐式转换
```

C++弱类型示例：

```cpp
int a = 10;
double b = 3.14;
void* p = &a;       // 指向 int
p = &b;             // 指向 double
```

JavaScript弱类型示例：

```javascript
let a = 10;
let b = "20";
let c = a + b;  // number -> string，隐式转换
```

## 编译型与解释型语言

两者的区别如下表所示：


| 类型     | 编译型                  | 解释型                  |
|----------|-------------------------|-------------------------|
| 执行方式 | 预编译成机器码          | 逐行解释执行            |
| 速度     | 更快                    | 较慢                    |
| 启动时间 | 较慢（需编译）          | 更快                    |
| 开发效率 | 较低（改动需重编译）    | 更高（改了立即运行）    |
| 跨平台   | 较差（需针对平台编译）  | 较好                    |
| 调试     | 较难                    | 更容易                  |
| 代表语言 | C/C++/Rust/Go           | Python/JS/PHP/Ruby      |

除此之外，还有使用了中间方法的编程语言如Java 和 C#。 源代码先编译成平台无关的字节码或中间语言，然后由虚拟机(JVM 或 CLR)在运行时将其解释或即时编译成机器码执行。这种方式保证了跨平台性和类型安全，同时在运行时可以进行优化，但性能略低于直接编译的原生语言。

## 栈内存(Stack)与堆内存(Heap)

栈内存和堆内存是程序运行时两种不同的内存管理区域：栈内存由系统自动分配和释放，存放局部变量和函数调用信息，访问速度快但空间有限，生命周期与作用域绑定；堆内存由程序员手动分配和释放(或由垃圾回收管理)，可存储大对象，空间灵活，但管理复杂且访问速度相对较慢。

| 特性 | 栈内存（Stack） | 堆内存（Heap） |
|------|----------------|----------------|
| 分配方式 | 系统自动分配 | 程序员手动分配或垃圾回收 |
| 释放方式 | 函数结束自动释放 | 程序员手动释放或垃圾回收 |
| 生命周期 | 随函数调用而定 | 跨函数，可灵活控制 |
| 访问速度 | 快（连续内存） | 较慢（需通过指针/引用） |
| 空间大小 | 较小，容易栈溢出 | 较大，适合大对象 |
| 优点 | 分配释放快，管理简单 | 可动态分配大块内存，生命周期灵活 |
| 缺点 | 空间有限，只能存放临时数据 | 分配释放慢，易内存泄漏或碎片化 |
| 典型用途 | 局部变量、函数参数、临时对象 | 大对象、动态数组、跨函数使用的对象 |
